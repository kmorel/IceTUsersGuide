\begin{Name}{3}{icetResetTiles}{Kenneth Moreland}{\IceT Reference}{\CFunc{icetResetTiles} -- clears out all tile definitions.}

  \mansection{Synopsis}

  \textC{\#include <IceT.h>}

  \begin{Table}{3}
    \textC{void }\CFunc{icetResetTiles}\textC{(}&\textC{void}&\textC{);}
  \end{Table}

  
  \mansection{Description}

  \IceT defines its display as a set of tiles.  \CFunc{icetResetTiles} will
  empty this set.  The set of tiles is filled again with calls to
  \CFunc{icetAddTile}.

  As a side effect, \CFunc{icetResetTiles} will also zero out the
  renderable window size (specified with state variables
  \CEnum{ICET\_PHYSICAL\_RENDER\_WIDTH} and
  \CEnum{ICET\_PHYSICAL\_RENDER\_HEIGHT}).  The size will be reset with
  calls to \CFunc{icetAddTile}.


  \mansection{Errors}

  None.


  \mansection{Warnings}

  None.


  \mansection{Bugs}

  None known.


  \mansection{Notes}

  As a rule, a call to \CFunc{icetResetTiles} should always be followed
  with one or more calls to \CFunc{icetAddTile}.  \CFunc{icetDrawFrame},
  \CFunc{icetCompositeImage}, and \CFunc{icetGLDrawFrame} will not work
  properly if no tiles are in existence.
